# Component Contracts: Modern GUI Redesign

**Feature**: 001-modern-gui-redesign
**Phase**: 1 — Interface Contracts
**Date**: 2026-04-18

These are the TypeScript prop interfaces for all new and redesigned components. They define
the public surface of each component — what callers must provide, what is optional, and what
the component promises to render.

---

## Shared Primitives (`components/ui/`)

Generated by shadcn/ui — see their source files for full prop contracts. Key usage contracts:

### Button variants
- `variant`: `"default" | "destructive" | "outline" | "secondary" | "ghost" | "link"`
- `size`: `"default" | "sm" | "lg" | "icon"`
- All buttons MUST have an accessible label (text content or `aria-label`)

### Badge variants
- `variant`: `"default" | "secondary" | "destructive" | "outline"`

## ConfirmDialog

```typescript
interface ConfirmDialogProps {
  open: boolean
  title: string
  description: string
  confirmLabel: string                      // shown on the confirm button
  cancelLabel?: string                      // default: "Cancel"
  onConfirm: () => void
  onCancel: () => void
  variant?: 'danger' | 'warning'           // default: 'danger'
  loading?: boolean                         // disables buttons + shows spinner
}
```

**Contract**: When `loading` is true, both buttons MUST be disabled. The dialog MUST NOT close
on backdrop click when `loading` is true.


**Contract**: MUST be rendered at the top of every page layout (above `<main>`). Renders null

## Cross-cutting Constraints

1. All interactive components on kid-facing screens MUST have a minimum tappable area of
   44×44px, enforced via `min-h-[44px] min-w-[44px]` Tailwind classes.

2. All components MUST accept `className?: string` for composition unless they are
   full-page layouts.

3. No component may hardcode color values — all colors MUST reference CSS custom properties
   from the design system tokens or Tailwind theme values mapped to those tokens.

4. Client components MUST be leaf-level — they MUST NOT import other `"use client"` components
   from Server Component files unless the import is at the usage site, not at the module level.
